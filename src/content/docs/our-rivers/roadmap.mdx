# Our Rivers — Phased Roadmap

This page shows where Our Rivers is going — 
version by version, from the first four rivers to a complete, 
live, AI-augmented database of every significant Indian river.

Each version has a clear scope, a clear goal, and clear success criteria.
Nothing is added to a version unless it is needed to prove that version's goal.

---

## Where we are now

<Badge text="Current: v0.01 — POC" variant="caution" />

The blueprint is written. The schema is confirmed. The folder structure is defined.
Four demo rivers are chosen. The first build is in progress.

## v0.01 — Proof of Concept

**Goal**: Prove the architecture works. Show the vision to the world.

| Dimension | Detail |
|-----------|--------|
| Rivers | 4 — Kaveri, Mahanadi, Seonath, Netravati |
| Zones | 2 live (Kaveri, Mahanadi) + 7 stubs |
| Data entry | 100% manual |
| DB | Local Postgres, queried at build time |
| Automation | None |
| Live data | Triveni-net component mounted, showing "coming soon" |
| Map | Placeholder only |
| Contributors | Team only |

**Done when**: All 4 rivers browsable, all 7 sub-pages rendering, 
all stub zones accessible, no broken pages, vision clearly communicable to anyone who visits.

## v1 — Complete Zone Coverage

**Goal**: All 9 zones live. Semi-automated data. Map component. Triveni-net connected.

<Tabs>
  <TabItem label="What changes">

| Feature | Detail |
|---------|--------|
| Rivers | ~30 total — all 9 zones have at least 2–3 rivers |
| Automation | Scripts pull from India-WRIS and CPCB to pre-fill periodic fields |
| Editorial gate | All automated data reviewed before publishing |
| Map component | Basic interactive river map per zone — course, tributaries, pollution colour-coding |
| Triveni-net | Live strip component connected for rivers with sensor coverage |
| Suggestion system | Popup/icon appears when `pollution_index: critical` — links to Pollution Academy and Get Involved |
| Search | All river fields indexed and searchable within Nadikosh |

  </TabItem>
  <TabItem label="What it looks like">

A reader visiting `/our-rivers/ganga-zone/` will see:
- A zone aggregate stats panel showing all Ganga basin rivers in the DB
- An interactive map of the Ganga basin with rivers coloured by pollution index
- A cards grid for every river in the zone — live `pollution_index` data, Triveni-net strip where available

A researcher visiting `/our-rivers/kaveri/health/` will see:
- A live reading strip at the top — DO, BOD, turbidity — powered by Triveni-net
- Weekly-updated pollution index with timestamp
- Known hotspot stretches and main pollution sources
- A cross-link suggestion: *"Pollution is critical on 2 stretches — explore Pollution Academy"
</TabItem>
</Tabs>

**Done when**: All 9 zones have real data, the map renders for at least 2 zones,
Triveni-net is connected for at least 1 river, and automated data flow is proven end-to-end.

## v2 — Intelligence & Scale

**Goal**: AI/ML data pipeline live. Advanced map. Full sensor coverage via Triveni-net.

| Feature | Detail |
|---------|--------|
| Rivers | Full zone coverage — all significant named rivers |
| AI/ML pipeline | Automated extraction from satellite imagery, news, research publications |
| Human validation | All AI-proposed data reviewed and approved before publishing |
| Advanced map | Flowing diagram with pollution hotspots, flow density, seasonal animation |
| Triveni-net | Live data for all rivers with available sensor coverage |
| Data freshness | Periodic fields updated daily for high-priority rivers |
| Cross-section intelligence | Suggestion system reads Triveni-net live data — alerts based on real-time readings |

<Aside type="note" title="The advanced map">
The v2 map is the most ambitious visual component in Nadikosh.  
It will be a purpose-built interactive diagram — not a Google Maps embed.  
It will show river courses, tributaries, pollution hotspots, flow density,  
and live sensor locations in a single, flowing visual.  
It will be the same component used across all river and zone pages,
configured by each river's data.  
Design and development of this component is a standalone workstream beginning in v1.
</Aside>
